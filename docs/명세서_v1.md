# [설계 문서] 자체 알고리즘 데이터 공급 API 시스템 (MVP)

## Ⅰ. 서비스 개요 및 비전

- **서비스명: 미정** / 자체 알고리즘 데이터 공급 API
- **한 줄 설명:** SWEA, 프로그래머스, LeetCode 등 공식 API를 제공하지 않는 플랫폼의 문제 데이터를 자체 데이터베이스(DB)화하여, 인증된 클라이언트에게 안정적으로 서빙하는 **문제 탐색 시간 단축용 코어 API 시스템**.
- **핵심 비전:** 외부 인프라 의존성을 완전히 배제하고 독자적인 API 공급 체계를 구축하며, 유저별 맞춤형 추천 이력 관리를 통해 지속 가능한 알고리즘 학습 생태계를 제공함.

## Ⅱ. 추진 배경 및 문제 정의

- **문제 정의 (Problem)**
    - **타 플랫폼의 API 부재:** 주요 알고리즘 플랫폼(SWEA, 프로그래머스, LeetCode)은 공식 API를 제공하지 않아 외부 클라이언트가 조건별 문제를 실시간으로 크롤링하거나 추천하는 로직을 구현하는 데 원천적인 제약이 존재함.
    - **탐색 비용 및 재접근성 저하:** 개발자가 매일 학습할 문제를 선별하는 데 소모되는 시간적 병목이 존재하며, 과거에 추천받았던 문제나 오답을 다시 찾아내기 위한 체계적인 이력 관리 수단이 부재함.
- **해결책 (Solution)**
    - **자체 API 엔진 및 Key별 독립 매핑 설계:** 직접 풀이하고 검증한 문제 데이터를 백엔드 인프라에 직접 주입(Direct Insert)하며, API Key별로 자정 이후 최초 호출 시 독립적인 문제를 랜덤 매핑 및 고정(멱등성 보장)함.
    - **추천 이력 기반의 대시보드 제공:** 유저별로 당일 배정받았던 문제 및 과거 추천 이력을 조회할 수 있는 인터페이스를 구축하여 복습 및 이력 추적 편의성을 극대화함.

## Ⅲ. 정보 구조도 (Information Architecture)

```java
자체 알고리즘 API 콘솔 (Root)
├── 1. 웹 콘솔 (Web Console UI - Vue)
│   ├── 1-1. 계정 인증 (로그인 / 회원가입)
│   ├── 1-2. API Key 관리 콘솔 (키 발급, 재발급, JSON 프리뷰)
│   └── 1-3. 추천 이력 페이지 (Dashboard - 과거 배정 문제 리스트 및 링크 연동)
└── 2. 코어 백엔드 API (Core BE Engine)
    ├── 2-1. 인증 가드 (API Key 유효성 검증 및 만료 키 차단)
    └── 2-2. 문제 공급 및 이력 엔드포인트
        ├── 2-2-1. 오늘의 문제 조회 API (API Key별 하루 한 문제 멱등성 보장)
        └── 2-2-2. 과거 추천 이력 리스트 조회 API (페이징 지원)
```

## Ⅴ. 운영 정책 및 비기능적 요구사항 (Non-functional Requirements)

- **호출 멱등성 및 당일 지정 보장:** 동일한 API 키로 당일 반복 호출이 발생하더라도 `DailyAllocation` 테이블에 기배정된 동일 레코드가 반환되어야 하며, 자정을 기점으로 정확히 다음 문제 추출 프로세스가 가동되어야 함.
- **이력 조회 성능 최적화 (F-08 연계):** 유저의 누적 이력이 증가함에 따라 대시보드 로딩 속도가 저하되는 것을 방지하기 위해, `DailyAllocation` 테이블 조회 시 유저 식별자 및 할당 일자 기준의 복합 인덱스(Composite Index)를 설정하고 페이징(Pagination) 처리를 필수 적용함.
- **만료 키 즉각 차단 (NF-02):** 소프트 딜리트 처리된 레거시 API 키로 인바운드 요청이 인입될 경우, 비즈니스 로직(Core Kernel)에 진입하기 전 인터셉터/필터 레이어에서 `401 Unauthorized` 예외를 즉각 반환하여 시스템 자원을 보호함.
